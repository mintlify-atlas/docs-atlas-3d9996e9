# Real Madrid RAG System

The Real Madrid RAG (Retrieval-Augmented Generation) system is an intelligent question-answering assistant that combines the power of **Azure AI Search** for document retrieval with **Azure OpenAI** for natural language generation. Built with .NET 8, this system provides accurate, context-aware responses about Real Madrid using retrieved source documents.

## What is RAG?

Retrieval-Augmented Generation (RAG) is an AI architecture that enhances large language models (LLMs) by:

1. **Retrieving** relevant documents from a knowledge base using semantic search
2. **Augmenting** the user's question with retrieved context
3. **Generating** accurate answers grounded in the retrieved information

This approach reduces hallucinations and ensures responses are based on verified source material.

## System Components

The Real Madrid RAG system consists of two main components:

### 1. RagConsole Application

The core RAG engine that processes questions and generates answers:

- **AzureSearchService**: Retrieves relevant documents from Azure AI Search index
- **AzureOpenAiService**: Generates natural language answers using GPT models
- **RagOrchestrator**: Coordinates the retrieval and generation pipeline

```csharp
// From Program.cs:5-22
var options = AzureOptionsLoader.Load();
var searchService = new AzureSearchService(options);
var openAiService = new AzureOpenAiService(httpClient, options);
var orchestrator = new RagOrchestrator(searchService, openAiService);
```

### 2. MCP Server

A Model Context Protocol server that provides coding standards to AI assistants:

- **GetCodingStandard**: Retrieve specific coding standards by topic
- **ListAvailableStandards**: Browse all available standards
- **SearchInStandards**: Search for keywords across all standards

<Info>
  The MCP server enables AI coding assistants to access Real Madrid project coding standards in real-time during development.
</Info>

## Response Format

Every query returns a structured response with:

```csharp
// RagResponse model from Models/RagResponse.cs:5
public sealed record RagResponse(string Answer, IReadOnlyList<RagSource> Sources);

public sealed record RagSource(string Id, string Source, string Content, double Score);
```

**Answer**: Natural language response generated by Azure OpenAI

**Sources**: List of retrieved documents with:
- `Id`: Unique document identifier
- `Source`: Document origin (e.g., article title, file name)
- `Content`: Relevant text chunk from the document
- `Score`: Search relevance score (0.0 to 1.0)

<Note>
  The system is designed to respond only with information from retrieved sources, explicitly indicating when context is insufficient.
</Note>
